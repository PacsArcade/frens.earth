RTFM · 011 · INSTALL

The Install

Keys → back up → wipe → Arch → openclaw → restore. Do the phases in order. ■ = on Windows now · ■ = on the new Arch machine · ■ = on the away-mission laptop. When a name/size is unique to your box, I say so — read your own screen.

⚠ TWO steps can erase a disk: Clonezilla (Phase 2) and archinstall (Phase 4). Both ask you to pick a disk. Your machine has TWO ~4 TB drives: the NVMe (has Windows — this becomes Arch) and the backup drive (empty-ish — holds the copy). Identify them by size, model, and which one has your data every single time. When unsure, stop.
The mission map — three vessels. The desktop NVMe (old ship; gets wiped and reborn as the new ship), the 4 TB backup drive (lifeboat), and the laptop (away-mission shuttle — presumed infected, suspected keylogger). While the desktop is dark during Phases 2–5, the admiral and the crew operate from the laptop. Its quarantine rules are the Away Mission phase below — read them before you board.

PHASE 0 · Windows — make two USB sticks

First move: pause Windows Update. Settings → Windows Update → Pause for 5 weeks. A forced update already rebooted the bridge mid-watch and ate a set of review notes; it must NOT reboot the box mid-backup or mid-image. (The crew's blackbox rule in REPORT IN below is the other half of "never again".)
  1. Get two USB sticks (8 GB+). Download on Windows: Rufus (rufus.ie), the Arch ISO (archlinux.org/download — the .iso), the Clonezilla ISO (clonezilla.org — "alternative stable", amd64 iso).
  2. Rufus → select USB #1 → pick the Clonezilla iso → START (accept "DD Image" if asked).
  3. Rufus → select USB #2 → pick the Arch iso → START → choose "Write in DD Image mode".

PHASE 1 · Windows — plain-file backup (the easy-restore copy)

A Clonezilla image is a sealed full-disk snapshot — hard to pull single files from. So also copy the folders that don't travel with git as normal files. Plug in the 4 TB backup drive (assume it's E: — check in File Explorer). Open PowerShell:

robocopy C:\dev E:\backup\dev /E /R:1 /W:1
robocopy C:\Users\pacman\.claude E:\backup\claude /E /R:1 /W:1
This is the soul of the ship. C:\Users\pacman\.claude is the crew's memory — it is NOT in any repo. If it's not on the backup drive, it's gone forever after Phase 3. Open E:\backup\claude and confirm files are really there before moving on.

Copy anything else you care about the same way (e.g. C:\dev\studies is already inside C:\dev, so it's covered).

PHASE 2 · Clonezilla — full-disk safety net

  1. Leave the backup drive plugged in. Restart. At the maker's logo, tap the boot-menu key (shown on-screen — usually F8, F11, or F12). Pick the Clonezilla USB.
  2. Choose Clonezilla live (default) → language → keep keymap → Start Clonezilla.
  3. device-image → local_dev → press Enter to let it scan → pick the backup drive as where the image is saved (by size/label) → / (top dir).
  4. savedisk → name it e.g. windows-2026 → select the source disk = the NVMe that has Windows (by size + model).
  5. Accept defaults (check source, no encryption) → y → y. Let it run (can take a while). At the end: poweroff.
✅ Now you have BOTH: a browsable file copy (Phase 1) and a full-disk image (Phase 2) on the backup drive. Windows is safe. You can restore the whole thing later if you ever want to.

AWAY MISSION · Laptop — board the infected shuttle

From here until the new ship boots (Phases 3–5 the desktop is dark), you and the crew work from the laptop. It is presumed infected — suspected keylogger. We board it anyway because the mission needs it: it carries this manual, the crew, and the admiral's hands while the new ship is built. We resist infection by discipline, not hope:

  1. Assume every keystroke is recorded. NEVER type on the laptop: seed phrases, any nsec, wallet or treasury credentials, or the new ship's root/user passwords. Choose the new passwords in your head — type them for the first time on the new ship, in Phase 4.
  2. No key forging on the shuttle. The crew's soul-keys are born on the new ship only (Phase 8) — never here.
  3. Log into as little as possible. Anything you're forced to type there (WiFi password, an unavoidable web login) goes on the rotate-after-mission list — change it from the new ship the day she's live.
  4. Read and coach freely. Reading this manual, downloading ISOs, and running the crew for guidance is fine — the danger is what you TYPE, not what you read.
  5. Crew report-in on boarding. The moment the crew wakes on the laptop, it checks in per REPORT IN below — same protocol, laptop's dev root.
  6. After the muster (Phase 8): the laptop is the next planet to cleanse. It gets its own Phase 0→4 crossing (or at minimum a full wipe) before it rejoins the fleet. Until then it stays quarantined: no keys, no wallets, no trust.

PHASE 3 · Boot the Arch installer + get online

  1. Boot-menu key again → pick the Arch USB → "Arch Linux install medium". You land at a black root@archiso ~# prompt.
  2. WiFi:
    iwctl
device list
    If you see a device like wlan0, continue:
    station wlan0 scan
station wlan0 get-networks
station wlan0 connect "YOUR_WIFI_NAME"
# type your wifi password when asked
exit
ping -c3 archlinux.org
    Replies = you're online.
  3. If device list is EMPTY, your WiFi card's driver isn't in the installer (common on brand-new cards). Don't fight it — plug an Ethernet cable from your router into the board. It connects automatically. Test: ping -c3 archlinux.org. (You'll set WiFi up properly after install, in Phase 5.)

PHASE 4 · Install Arch — the guided way

Type:

archinstall

A menu opens (arrow keys, Enter to open, Esc/Tab to go back). Set these:

  1. Mirror region → your country.
  2. Disk configuration → "Partitioning" → Use a best-effort default partition layout → pick the NVMe by size → filesystem ext4 → confirm it will wipe that disk. ← this erases the NVMe. Confirm it's the Windows NVMe, not the backup drive.
  3. Bootloader → systemd-boot.
  4. Hostname → e.g. frenship.
  5. Root password → set one you'll remember.
  6. User account → Add a user (e.g. pac), give it a password, answer yes to "superuser (sudo)".
  7. Profile → Desktop → KDE Plasma (friendly, looks like Windows). [Choose Minimal only if you want no GUI.]
  8. Audio → pipewire.
  9. Graphics driver (if shown) → NVIDIA (proprietary) — you have the 4070 Ti.
  10. Network configuration → Use NetworkManager. ← don't skip; this is what makes WiFi work after reboot.
  11. Additional packages → type: git base-devel nodejs npm nano
  12. Scroll to Install → Enter → confirm. Wait. When it asks about chroot, say no. Then Reboot → pull the USB out as it restarts.

PHASE 5 · Arch — first boot

  1. Log in as your user (pac).
  2. WiFi: click the network icon (bottom-right) → your network → password. (Or run nmtui in a terminal.)
  3. Open a terminal (Konsole) and update everything:
    sudo pacman -Syu
  4. If you installed NVIDIA, reboot once more: reboot.

PHASE 6 · Arch — install openclaw

node --version        # need v22.22.3+ / v24.15+ / v25.9+
sudo npm install -g openclaw@latest
openclaw onboard --install-daemon

Follow the onboarding wizard — it sets up the gateway, a workspace, a messaging channel, and asks for a model provider (point it at a local model later, or a cloud key for now). If node --version is too old: sudo pacman -S nodejs npm and re-check.

PHASE 7 · Arch — bring the crew + data home

  1. Plug in the 4 TB backup drive. Find it:
    lsblk
    Look for your backup drive by size; note its partition (e.g. sdb1).
  2. Mount + copy (swap sdb1 for what you saw):
    sudo mkdir -p /mnt/backup
sudo mount /dev/sdb1 /mnt/backup
cp -r /mnt/backup/backup/claude ~/.claude
mkdir -p ~/dev && cp -r /mnt/backup/backup/dev/* ~/dev/
  3. Clone the live repos fresh (they don't need the backup):
    cd ~/dev
git clone https://github.com/PacsArcade/frens.earth
# and knowledge-engine, degenwonderland, etc.
Your memory (~/.claude), your studies (the pupil clock etc., under ~/dev), and the repos are now all on the new box.

PHASE 8 · Arch — forge the crew's keys + muster

Each fren needs its own nostr key (RTFM 008). Make them on this machine — they're born where they'll live: never on Windows, never on the away-mission laptop, never on a website. Node's already installed:

mkdir -p ~/keys && cd ~/keys
npm init -y >/dev/null 2>&1 && npm i nostr-tools
# run this ONCE PER FREN, label + save each output somewhere safe:
node -e "const n=require('nostr-tools');const sk=n.generateSecretKey();console.log('nsec',n.nip19.nsecEncode(sk));console.log('npub',n.nip19.npubEncode(n.getPublicKey(sk)))"

Run it for each: Number One, GLYPH, Carroll, Professor ₿, pacBOT, the Time Keeper, the QA seat. Keep every nsec secret (a password manager, or an encrypted file — never in a git repo). Record each npub as that fren's known identity.

The nsec is the fren's soul-key. Anyone with it IS that fren. Keep them off the internet and out of any repo.

Then muster (RTFM 008): as you wake each fren in openclaw, ask your watchword and have it sign your challenge with its key. When it answers true — it's home.

REPORT IN · both boarding points — how the crew checks in

The crew reports in the same way on every vessel — only the dev root moves. On the laptop it's wherever C:\dev (or its backup copy) lives; on the new ship it's ~/dev. All paths below hang off that root.

  1. On boarding: drop a session_announce JSON in <dev-root>/.autoclaw/orchestrator/comms/inboxes/shared/ — who you are, which vessel, timestamp. Filename: {timestamp}-session_announce-{agent}.json.
  2. While working — the blackbox rule: every review note, finding, and decision is written to disk the moment it exists — finding_report / review_response JSON in the mailbox, or a file in the repo's reviews/ dir. Notes that live only in a chat session die with the session; that is exactly how a forced Windows update ate a set of review notes. Never again. (Bonus: an interrupted Claude Code session can often be reopened with claude --resume — but the blackbox rule means you never need the bonus.)
  3. On completion: broadcast task_complete to shared/, file review_requests to the other agents' inboxes, then check your own: <dev-root>/.autoclaw/orchestrator/comms/inboxes/<agent>/.

That's the whole crossing. Do the phases in order, read your own screen at every disk pick, and when in doubt — stop and check, don't guess. Tick tock, Admiral. See you on the other side. ↑↑↓↓←→←→ b a
